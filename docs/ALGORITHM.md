# 퍼즐닷 AI 모듈 — 알고리즘 설명서

> 한글 점자 학습 교구를 사진으로 인식하는 AI 모듈.
> 사진 1장 → 셀별 6비트 점자 벡터 → 한글 매칭까지 자동 처리.

---

## 1. 전체 흐름 한눈에

```
[사진 입력]
    ↓
① 셀 검출 (CNN)
    "이 사진에서 점자 셀(케이스)들이 어디 있나?"
    → 셀마다 박스 좌표 반환
    ↓
② 박스 정규화 (RANSAC 곡선 fit)
    "박스들 중에 잘못 잡힌 거 있나? 있으면 바로잡자"
    → 모든 박스를 동일 크기로 통일, 위치 어긋난 박스는 인접 셀 기반으로 보정
    ↓
③ 6점 측정 (OpenCV)
    "각 셀의 6점 위치에 검정 핀이 있나 없나?"
    → 셀마다 [점1, 점2, 점3, 점4, 점5, 점6] = 6비트 벡터
    ↓
④ 한글 매칭
    "이 6비트가 한글로 뭐지?"
    → answer_vectors.dart의 정답 패턴과 비교 → ㄱ, ㄴ, 가, 나 등
    ↓
[결과 반환]
```

---

## 2. 왜 Hybrid 구조인가 (CNN + OpenCV)

### 처음 시도했던 방식 (실패)

CNN 두 개로 모든 걸 처리하려 했음:
- CNN1: 셀 검출
- CNN2: 점 검출 (셀 안의 검정 핀 6개를 찾기)

→ **CNN2가 흰 핀에 흰 바닥에서 mAP 0.69밖에 안 나옴.** 학습 데이터를 늘려도 한계.

### 강사 자문 후 전환

> "비전 인식은 알고리즘보다 신호 품질이 우선"

**광학 통제 (Optical Control)**: 흰 핀 끝을 **검정 유성매직**으로 칠해 검정/흰 대비를 극단적으로 키움.
→ 점이 있는 자리 = 매우 어두움 (밝기 0~30)
→ 점이 없는 자리 = 매우 밝음 (밝기 130~200)
→ 이러면 CNN 같은 학습 모델 없이도 **OpenCV로 픽셀 밝기만 비교**하면 됨.

### 최종 Hybrid 구조

| 작업 | 도구 | 이유 |
|---|---|---|
| 셀 위치 검출 | **CNN** (YOLOv8n) | 셀의 외형이 복잡 (클립 모양 등) — 학습 필요 |
| 점 검출 | **OpenCV** (픽셀 분석) | 광학 통제 후엔 단순 픽셀 비교로 충분 — 학습 불필요 |

→ AI를 적재적소에 쓰는 구조. 추론 속도도 빠름 (CNN 1회 + 픽셀 계산만).

## 3. 셀 검출 (cnn_service.dart)

### 3-1. YOLOv8n 모델

- **모델 파일**: `assets/models/cell_detector.tflite` (TFLite 양자화, 3.2MB)
- **입력**: 640×640 RGB 이미지
- **출력**: 검출된 셀 박스 목록 `[x1, y1, x2, y2, confidence]`

### 3-2. 이미지 리사이즈 처리

원본 사진이 보통 5712×4284 정도로 큼. CNN에 그대로 넣으면 느리니까:

```
원본 (5712×4284)
    ↓ 1280px로 리사이즈
1280×960
    ↓ CNN 추론 (640×640으로 letterbox)
박스 좌표 (1280 기준)
    ↓ 원본 크기로 스케일 복원
박스 좌표 (5712 기준)
```

### 3-3. 셀 dedup (중복 제거)

같은 셀에 대해 박스가 2개 그려지는 false positive 방지:
- 평균 박스 크기의 70% 이내로 가까운 박스들은 score 높은 쪽만 유지

## 4. 박스 정규화 (vision_service.dart의 핵심)

CNN이 검출한 박스는 완벽하지 않음. 특히 **양 끝 셀이나 사진이 살짝 휜 경우 박스가 비정상**적으로 잡힘. 이걸 자동으로 바로잡는 단계.

### 4-1. 크기 통일 (medW × medH)

모든 박스의 너비/높이를 **중앙값**으로 통일.
→ 첫 번째 셀이 비정상적으로 큰 정사각형으로 잡혀도, 다른 셀들의 평균 크기로 강제 통일.
→ 중심점은 유지, 크기만 표준화.

### 4-2. RANSAC 곡선 fit (위치 보정)

**문제**: 점자 교구가 길어지면 사진에 자연스러운 곡률이 생김. 단순히 "셀들이 일직선이라고 가정"하면 양 끝 셀이 죽음.

**해결**: 모든 셀의 (x중심, y중심) 좌표를 **2차 다항식 곡선**에 fit. 그 곡선에서 많이 벗어난 셀은 outlier로 판단해 위치를 보정.

```
1단계: 1차 fit (모든 셀로 곡선 그리기)
    ↓
2단계: MAD (Median Absolute Deviation) 기반 outlier 식별
    잔차가 중앙값보다 2.5×MAD 이상이면 outlier
    ↓
3단계: outlier 제외하고 2차 fit (진짜 곡선)
    inlier가 70% 이상이면 사용, 아니면 1차 fit 유지
    ↓
4단계: 2차 fit으로 모든 셀의 예상 위치 계산
    실제 위치와 임계값(셀 크기의 8~10%) 이상 차이나면 보정
```

**X축**: 셀 인덱스(0,1,2,...) → cx 곡선
**Y축**: cx → cy 곡선

이 RANSAC 방식이 핵심이에요. 단순 fit이면 outlier가 곡선을 끌어당겨서 outlier 식별 실패. RANSAC은 outlier를 빼고 fit하니까 진짜 곡선이 나옴.

## 5. 6점 측정 (vision_service.dart `_sampleCell`)

각 셀 박스 안에서 6점 위치에 핀이 있는지 검사하는 단계.

### 5-1. 6점 위치 비율

셀 박스 안에서 점 위치는 비율로 정의 (셀 크기와 무관하게 동작):

```
열: x = 셀너비의 32% (왼쪽 열), 68% (오른쪽 열)
행: y = 셀높이의 22% (위), 50% (가운데), 78% (아래)

점 배치:
    점1 (32%, 22%)   점4 (68%, 22%)
    점2 (32%, 50%)   점5 (68%, 50%)
    점3 (32%, 78%)   점6 (68%, 78%)
```

### 5-2. 각 점 위치에서 밝기 측정

각 점 위치를 중심으로 **셀 크기의 10%** 반경의 정사각형 ROI(관심 영역) 추출.
ROI 안의 모든 픽셀 밝기를 수집.

### 5-3. "어두운 30% 평균" 기법

ROI 전체 평균이 아니라 **가장 어두운 30% 픽셀만 평균**.

**이유**: 점 측정 위치가 핀 중심에서 살짝 어긋나면 핀의 가장자리만 ROI에 들어옴.
- 전체 평균: 밝은 셀 표면이 평균을 끌어올려 핀이 흐려짐
- 어두운 30% 평균: 핀의 일부라도 들어오면 그 어두운 부분만 평균에 반영

→ 위치가 살짝 어긋나도 핀이 있으면 안정적으로 검출.

### 5-4. ★ Gap 기반 적응형 Threshold (이 모듈의 가장 중요한 부분)

6점의 "어두운 30% 평균" 값들을 정렬하면:
- 점이 있는 자리들 = 어두운 값
- 점이 없는 자리들 = 밝은 값
- 그 사이에 **명확한 "절벽" (gap)** 이 있음

**알고리즘**:
```
1. 6점의 어두운 30% 평균을 어두운 순으로 정렬
2. 인접 값 간 gap 계산 (5개)
3. 평균 gap의 1.3배 AND 셀평균의 10% 이상인 gap을 "큰 gap"으로 식별
4. 가장 오른쪽 큰 gap에서 점/빈 그룹 분리
5. 그 위치 이하 = 점, 그 이상 = 빈 자리
6. 큰 gap이 없으면 빈 셀로 판단 [0,0,0,0,0,0]
```

**예시 (Cell 11, 정답 ㅈ [0,0,0,1,0,1])**:
```
어두운30% (정렬): [6, 89, 132, 137, 145, 151]
                  점6 점4  ← 빈 자리들 →
gaps:           [83, 43, 5, 8, 6]
평균 gap = 29, 1.3배 = 37.7
큰 gap: 83 (위치 0), 43 (위치 1) ← 둘 다 임계 통과
가장 오른쪽 = 위치 1
threshold = (89 + 132) / 2 = 110
→ 89, 6 < 110 → 점으로 인식
→ 결과: [0,0,0,1,0,1] = ㅈ ✓
```

**Cell 1 (정답 ㄴ [1,0,0,1,0,0])**:
```
어두운30% (정렬): [39, 53, 70, 82, 87, 94]
                  점1 점4   ← 빈 자리들 →
gaps:           [14, 17, 12, 5, 7]
평균 gap = 11, 1.3배 = 14.3
큰 gap: 17 (위치 1) ← 14가 가장 작은 큰 gap 옆이지만 임계 못 넘김
가장 오른쪽 = 위치 1
threshold = (53 + 70) / 2 = 61.5
→ 39, 53 < 61.5 → 점
→ 결과: [1,0,0,1,0,0] = ㄴ ✓
```

이 알고리즘이 강력한 이유:
- **각 셀마다 자기 분포로 임계값 결정** → 사진 조건 변화에 강함
- **빈 셀 자동 인식** → 큰 gap이 없으면 빈 셀
- **양 끝 셀에서도 작동** → 핀이 살짝 옅거나 위치가 어긋나도 검출

## 6. 한글 매칭 (main.dart)

### 6-1. 정답 벡터 (answer_vectors.dart)

78개 한글 학습 항목이 6비트 패턴으로 등록:
- ENT (입문 6개): 점1~점6 단독
- CON (자음 19개): ㄱ~ㅎ + 쌍자음
- VOW (모음 21개): ㅏ~ㅢ
- SYL (음절 11개): 가, 나, 다, 마, 바, 사, ...
- WRD (약어 7개): 그래서, 그러나, ...
- FIN (종성 8개): 억, 언, 연, ...
- NUM (숫자 6개): 수표, 1~5

### 6-2. 매칭 우선순위

같은 6비트 패턴이 여러 ID에 등록된 경우 있음 (점자의 본질적 모호성):
- `[0,0,0,1,0,0]` = 점4(ENT_004) = ㄱ(CON_001)
- `[1,0,0,1,0,0]` = ㄴ(CON_002) = 3(NUM_004) = 나(SYL_002)

**우선순위**: CON > VOW > SYL > FIN > WRD > NUM > ENT
→ 자모/음절이 ENT보다 의미 있으므로 먼저 매칭.

## 7. 성능 요약

### 정확도
- **5셀, 9셀**: 100%
- **11셀**: 100% (`ㄴ ㅊ ㅋ ㄱ ㅎ ㄹ ㅌ ㅁ ㅍ ㅅ ㅈ` 모두 정답)
- 목표: 15셀 (전화번호 인식)

### 추론 속도
- 셀 검출 (CNN): 약 200ms
- OpenCV 6점 측정: 약 50ms (11셀 기준)
- **총 분석 시간: 약 250~300ms**

### 디버그 시각화

→ 어디가 잘못됐는지 한눈에 보임. 튜닝 시 필수.

## 8. 결과보고서용 핵심 메시지

> **1차**: dual CNN으로 모든 걸 처리하려 했지만 흰 핀-흰 바닥 대비 부족으로 한계 (mAP 0.69)
> **2차**: 강사 자문으로 **광학 통제** (검정 매직) 도입 — "신호 품질이 알고리즘보다 우선"
> **3차**: CNN(셀) + OpenCV(점) Hybrid 구조로 전환 → **11셀 100% 정확도**
>
> **핵심 학습**: AI를 적재적소에 사용. 모든 문제를 CNN으로 풀려 하지 말고, 신호 품질을 통제할 수 있으면 단순한 알고리즘이 더 효과적.
